# Security Policy

The 402Kit team takes security seriously. We appreciate your efforts to responsibly disclose your findings, and we will make every effort to acknowledge your contributions.

## Reporting a Vulnerability

**DO NOT** use public channels for security reports.

Please report security vulnerabilities to **[security@402kit.dev](mailto:security@402kit.dev)**. A PGP key for encrypted communication will be provided here in the future.

Please include the following details in your report:

- Affected package(s) and version(s)
- A proof-of-concept (PoC) or detailed steps to reproduce
- The potential impact of the vulnerability

## Scope

This policy applies to the following packages within this repository:

- `@402kit/core`
- `@402kit/client`
- `@402kit/server`
- `@402kit/adapter-x402`
- `@402kit/adapter-mock`
- `@402kit/entitlement`

Third-party dependencies are not covered by this policy.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| `1.x`   | :white_check_mark: |
| `< 1.0` | :x:                |

## Response SLA

We will make a best effort to meet the following response timelines for security reports:

- **Acknowledgment**: Within 2 business days
- **Initial Assessment**: Within 5 business days
- **Fix & Release**: Depends on severity, but we will keep you informed of our progress.

## Disclosure Policy

We coordinate with the reporter to fix the vulnerability before making a public disclosure. Once a fix is released, we will publish a security advisory on GitHub and credit the reporter.

## Hall of Fame

Contributors who responsibly disclose security issues will be acknowledged here (with permission).
